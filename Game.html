Prompt Engineering: Before vs After

Core 1: Role Prompting

Before: "Make a game."

After: "You are a senior front-end game developer. Build a browser Whac-A-Mole game."

Result: Cleaner architecture and clearer variable naming.

Core 2: Constraint Prompting

Before: "Add scoring and timing."

After: "Use a 3x3 grid, 30-second timer, score +1 per hit, vanilla HTML/CSS/JS only."

Result: Output matched required specs without extra libraries.

Core 3: Few-Shot Structure

Before: "Style the interface nicely."

After: "Follow this format: HUD cards on top, board in middle, controls below."

Result: Predictable layout and easier UI validation.

Extra Technique: Iterative Refinement

Before: One-shot prompt only.

After: "Now add a Golden Mole worth +3 points and a visible streak counter."

Result: New feature integrated successfully into gameplay loop.
